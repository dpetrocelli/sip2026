Observaciones transversales

Dashboard NDJSON — patrón recurrente
  • 3 de 4 grupos dejaron scraper-overview.ndjson como placeholder o index-pattern incompleto
  • El proceso es directo: crear el dashboard en Kibana → Saved Objects → Export con "Include related objects" → guardar como NDJSON
  • El import via POST /api/saved_objects/_import con kbn-xsrf: true lo tienen varios grupos — solo falta el archivo correcto
ADR 0007 — Loki
  • 2 de 4 grupos no entregaron el ADR 0007 (Proxy Team, CFZ++)
  • La implementación es correcta en ambos — es un gap de documentación puro
  • Con el README y la config ya existentes, se escribe en 30 minutos
ADR 0012 — bloqueante P4
  • Systeam y CFZ++: ADR 0012 no existe → P4 = 0
  • Proxy Team: ADR 0012 existe pero ~1,100 palabras (mínimo: 1,500) → P4 = 0
  • Solo Group ONE entregó P4 con ADR completo (~3,850 palabras)
Evidencia fotográfica
  • Patrón común: ANTES/DESPUES screenshots de formato de log, pero sin Grafana Explore, sin queries ejecutadas, sin dashboard con datos
  • Los screenshots son la diferencia entre un 8.0 y un 9.5 en P1 y P2
OpenSearch en ADR 0009
  • CFZ++ nombra OpenSearch explícitamente (10/10)
  • Group ONE y Systeam mencionan ELv2 pero sin nombrar OpenSearch como fork (9/10)
  • Proxy Team menciona "alternativas OSS" sin nombrar OpenSearch (9/10)
